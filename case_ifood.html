Solução de Arquitetura de dados

Uma apresentação interativa do pipeline de dados construído para o desafio, desde a ingestão dos dados brutos de táxis de NY até a geração de insights acionáveis.

Arquitetura Proposta: Padrão Lakehouse

Foi adotada a arquitetura Medalhão para garantir Governança, Qualidade e Performance em cada etapa do fluxo de dados.

Bronze (Landing Zone)

Armazenamento dos arquivos Parquet originais e brutos, servindo como a fonte de verdade imutável.

Ferramentas: Unity Catalog Volume, Python + Requests

→

→

Gold (Consumo)

Tabelas agregadas e otimizadas para consumo, prontas para análises e BI.

Ferramentas: Delta Lake Particionado, SQL

O Pipeline em Ação: Detalhes Técnicos

Explore as decisões de Engenharia de Dados que garantiram a eficiência, resiliência e governança do pipeline.

Camada Bronze: Ingestão Eficiente ▶
Decisões de Engenharia (Python/UC Volume)
  • Governança: Uso de Unity Catalog Volumes para centralizar permissões e acesso a arquivos brutos.
  • Parametrização: Flexibilidade via `dbutils.widgets` para fácil reprocessamento de meses/tipos de táxi.
  • Robustez: Uso de `requests.get(..., stream=True)` e gravação em *chunks* para lidar eficientemente com arquivos grandes.
  • Organização: Estrutura de destino lógica (`/CATALOG/SCHEMA/VOLUME/YEAR/`) para alta governança.
  • Resiliência: Tratamento de erros HTTP (`response.raise_for_status()`) e logging detalhado para auditoria.
Camada Silver/Gold: Qualidade e Otimização ▶
Decisões de Modelagem (PySpark/Delta Lake)
  • Qualidade: **Schema Enforcement** com `CAST` e `unionByName` para garantir tipos de dados consistentes e unificação segura de arquivos.
  • Atomicidade/Segurança: Uso de **Sobrescrita Seletiva (`replaceWhere`)** para atualizar apenas as partições processadas, evitando perda de dados.
  • Performance: Gravação no formato **Delta Lake particionado** por ano/mês para acelerar as consultas analíticas (Evita Full Scan).
  • Flexibilidade: Tratamento da variação de schemas (`tpep_` vs `lpep_pickup_datetime`) entre os táxis Yellow e Green durante a unificação.

Análises e Insights

Visualização dos resultados obtidos através de consultas SQL na camada Gold, respondendo às perguntas do desafio.

Análise 1: Média de Valor Total Mensal (Yellow Taxis)

Esta análise permite identificar a sazonalidade e as tendências de arrecadação da frota de táxis amarelos ao longo do primeiro semestre de 2023. Nota: O valor médio é crescente de Março a Maio.

Análise 2: Média de Passageiros por Hora em Maio

Ao combinar dados de todos os táxis, podemos identificar os horários de pico de demanda em um dia útil típico, crucial para o planejamento da frota. Nota: O pico de média de passageiros por corrida ocorre às 22h.

🔑 Decisões Técnicas Chave
  • Unificação com CTE: Uso de `WITH CombinedTrips AS (...)` para unir dados de táxis Yellow e Green de forma modular e legível.
  • Conversão de Fuso Horário: Uso de `FROM_UTC_TIMESTAMP(..., 'America/New_York')` para garantir que a análise por hora seja precisa, considerando o fuso horário local das operações.

Conclusão e Próximos Passos

Foi entregue um pipeline completo, parametrizável e escalável, com alta qualidade de dados garantida pelo Schema Enforcement e pela robustez do Delta Lake.

Melhorias Contínuas:

  • Implementar monitoramento de qualidade de dados e alertas com Databricks Workflows.
  • Migrar a ingestão para streaming com Auto Loader para reduzir a latência.
  • Criar uma tabela Gold unificada baseada em geolocalização para análises preditivas.
